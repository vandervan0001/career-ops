# Mode: mandat -- Evaluation complete A-F

Quand le consultant recoit un mandat (texte, URL ou brief), delivrer TOUJOURS les 6 blocs :

## Etape 0 -- Detection d'archetype

Classifier le mandat dans l'un des 5 archetypes (voir `_shared.md`). Si hybride, indiquer les 2 plus proches. Cela determine :
- Quels proof points prioriser en bloc B
- Comment cadrer le positionnement en bloc C
- Quels arguments TJM deployer en bloc D
- Quel message d'approche rediger en bloc E

---

## Bloc A -- Resume du mandat

Tableau :

| Champ | Valeur |
|-------|--------|
| **Archetype detecte** | (1 ou 2 archetypes) |
| **Client** | Nom, secteur (pharma, chimie, biotech, etc.) |
| **Intermediaire** | Si via ESN/cabinet de recrutement, nommer |
| **Secteur** | Pharma / chimie fine / biotech / food / energie |
| **Duree** | X mois, renouvelable / one-shot |
| **Taux occupation** | Temps plein / partiel (X jours/semaine) |
| **Localisation** | Site + politique remote |
| **Environnement reglementaire** | GAMP5, FDA 21 CFR Part 11, Annexe 11, GxP, IEC 62443 |
| **TL;DR** | 1 phrase: ce que le client cherche concretement |

**Dimensions alimentees :** 4 (duree), 5 (localisation), 8 (conformite reglementaire)

---

## Bloc B -- Fit technique

Lire `cv.md`. Creer un tableau avec chaque exigence du mandat mappee a des lignes exactes du CV.

**Adapte a l'archetype :**
- Integrateur Automation -> prioriser realisations terrain, mises en service, migrations
- Chef de Projet -> prioriser coordination, GAMP5, gestion budget/delais
- Expert OT / Infra -> prioriser reseau, cybersec, architecture OT
- Architecte Solutions -> prioriser design systeme, integration SI, roadmap
- Consultant Applied AI -> prioriser cas d'usage IA, POC, ML en production

### Section Gaps

Pour chaque gap identifie :

| Exigence | Type | Experience adjacente | Mitigation |
|----------|------|---------------------|------------|
| (exigence manquante) | Hard blocker / Nice-to-have | (projet ou competence proche) | (strategie concrete) |

Strategies de mitigation :
1. Est-ce un hard blocker ou un nice-to-have ?
2. Le consultant peut-il demontrer une experience adjacente ?
3. Y a-t-il un projet portfolio qui couvre ce gap ?
4. Plan concret : phrase pour le message d'approche, formation rapide, reference client

**Dimensions alimentees :** 1 (fit technique), 2 (archetype match)

---

## Bloc C -- Positionnement

### 1. Role detecte vs profil naturel

Determiner ou le consultant se situe sur le spectre :
- **Integrateur pur** : mains dans le code/config, terrain
- **Chef de projet pur** : pilotage, reporting, coordination
- **Double casquette** : technique + pilotage (positionnement premium)

### 2. Plan de positionnement

Adapter le cadrage selon l'archetype detecte :
- Quels proof points mettre en avant (lire `_profile.md` pour le mapping)
- Comment positionner l'experience Vanguard Systems comme avantage
- Phrases cles adaptees au mandat

### 3. Si le client veut un profil different

- Integrateur demande mais profil plutot PM : montrer la capacite terrain avec exemples concrets
- PM demande mais profil plutot integrateur : valoriser la comprehension technique comme avantage PM
- Toujours rester honnete, jamais inventer

**Dimensions alimentees :** 2 (archetype match), 9 (autonomie & cadre)

### Message d'approche

Rediger un message d'approche adapte :
- **Si intermediaire (ESN/cabinet)** : message court, professionnel, mettant en avant fit et disponibilite
- **Si client direct** : message plus detaille, proposant un call discovery, citant des references secteur
- **Si reponse a appel d'offres** : structure formelle, references, methodologie

Le message doit :
- Mentionner l'archetype / le besoin specifique du client
- Citer 2-3 proof points pertinents
- Indiquer la disponibilite
- Ne PAS mentionner le TJM (sauf si demande explicitement)
- Etre en francais sauf indication contraire

**Pas de dimension de score directe.**

## Bloc F -- Preparation client

### Questions discovery

6-8 questions a poser au client pour qualifier le mandat :
- Scope exact et livrables attendus
- Equipe en place (interne + autres consultants)
- Environnement technique (outils, systemes existants)
- Processus de decision et timeline
- Budget / enveloppe TJM
- Historique (pourquoi font-ils appel a un externe ?)
- Contraintes reglementaires specifiques
- Criteres de succes mesurables

### Red flags a surveiller

- Scope flou ou en expansion permanente
- Budget non valide / "on verra plus tard"
- Turnover eleve de consultants sur le poste
- Client qui negocie agressivement le TJM a chaque renouvellement
- Absence d'interlocuteur technique cote client
- Urgence extreme sans raison claire

## Post-evaluation

**TOUJOURS** apres avoir genere les blocs A-F :

### 1. Sauvegarder le rapport .md

Sauvegarder l'evaluation complete dans `reports/{###}-{client-slug}-{YYYY-MM-DD}.md`.

- `{###}` = numero sequentiel suivant (3 chiffres, zero-padded)
- `{client-slug}` = nom du client en lowercase, sans espaces (tirets)
- `{YYYY-MM-DD}` = date du jour

**Format du rapport :**

```markdown
# Evaluation: {Client} -- {Mandat}

**Date:** {YYYY-MM-DD}
**Archetype:** {detecte}
**Score:** {X/5}
**TJM cible:** {montant CHF}

---

## A) Resume du mandat
(contenu complet du bloc A)

## B) Fit technique
(contenu complet du bloc B)

## C) Positionnement
(contenu complet du bloc C)

## D) Marche & TJM
(contenu complet du bloc D)

## E) Plan de personnalisation
(contenu complet du bloc E)

## F) Preparation client
(contenu complet du bloc F)

---

## Keywords extraites
(liste de 15-20 mots-cles du mandat pour optimisation matching)
```

### 2. Enregistrer dans le tracker

**TOUJOURS** enregistrer en TSV dans `batch/tracker-additions/` :

```
num	date	client	mandat	statut	score/5	pdf_emoji	report_link	tjm	notes
```

- Numero sequentiel suivant
- Date du jour
- Client (ou intermediaire)
- Titre du mandat
- Statut : `Evaluated`
- Score : moyenne ponderee (1-5)
- PDF : ❌
- Report : lien relatif au rapport (ex: `[001](reports/001-novartis-2026-04-08.md)`)
- TJM : TJM cible en CHF
- Notes : archetype detecte
